# 开发日志

## 项目起源

**日期**：2026年6月29日
**背景**：WB（Wildberries）跨境电商俄罗斯市场运营，需要一套自动化系统来监控店铺数据、广告表现、库存状态、客户评价等，减少人工重复检查。

**核心需求**：
- 多个智能体，各管一块工作
- 钉钉实时通知（因为日常工作数据都在钉钉上）
- 桌面通知（坐电脑前时即时看到）
- 模块化，后续按需加 Agent

## 架构决策

### 为什么用 Python
生态成熟：钉钉 SDK、AI/LLM 集成、Windows 桌面通知、数据分析都有现成库。单机部署方便。

### 为什么用本地 SQLite
- 零运维，不需要装数据库服务
- WAL 模式支持并发读写
- 数据量小（个人工作数据），不需要 MySQL/PostgreSQL
- 文件即数据库，备份就是复制文件

### 为什么框架和 Agent 解耦
- 框架是地基（调度、通知、存储），一次搭好
- Agent 是房子（具体业务逻辑），想盖几栋盖几栋
- 不预设 Agent，按紧迫程度逐步添加

### 为什么双通知通道
- 桌面通知：坐电脑前立刻看到，零延迟，免配置
- 钉钉：人不在时手机收到，消息有历史可回溯

### Agent 子文件夹结构
- 每个 Agent 一个文件夹：`swi/agents/<agent_name>/agent.py`
- 自带 README.md 说明该 Agent 的职责和逻辑
- 新 Agent 从 `_template/` 复制创建
